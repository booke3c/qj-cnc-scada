技術參考資料（點擊展開）
OPC UA 開發鐵則（來源：Heidenhain 官方手冊 01/2026）

以下規則直接來自 HEIDENHAIN OPC UA Information Model 手冊（214頁），為開發必須遵守的硬性約束。

連線與安全
Namespace 動態查詢ns= 索引禁止硬編碼，每次連線用 session.getNamespaceIndex() 取得。
初次連線策略先用 SecurityMode=None + Username/Password 驗通，再換 Basic256Sha256 + X.509 憑證。
憑證雙向信任控制器端須在 TNC OPC UA Assistant 信任客戶端憑證，缺一不可。
等 NCIsAvailableNC 啟動後需等 Machine.State.CurrentState = NCIsAvailable 才操作。
FileSystem 程式傳輸（Chapter 9）
程式傳輸協議走 OPC UA FileSystem（Chapter 9），非 FTP / LSV2。
禁止遞迴瀏覽FileSystem 龐大且動態，遞迴瀏覽嚴重影響效能，手冊明確警告。
NodeId 是動態的File / Directory 的 NodeId 動態分配，不可快取或硬編碼，每次操作前 Browse 取得。
Handle 必須 CloseOpen() 後無論成功或失敗都必須呼叫 Close()，否則後續操作失敗。
PLC 變數（Chapter 10）
需機台側設定PLC 變數透過 CfgOpcUaPlcVar(134100) 開放，未設定則不出現在 Address Space。
PLC 重新編譯ManufacturerExtensions 節點全部重建，NodeId 可能改變，客戶端需重新 Browse。
IPC 安裝部署流程（10 步驟）
1

安裝 Node.js + pnpm

corepack enable && corepack prepare pnpm@latest --activate
2

複製專案至 IPC

git clone <remote> C:\CNC_SCADA && cd C:\CNC_SCADA && pnpm install
3

設定環境變數

DATABASE_URL="file:./dev.db" / PORT=3001 / MACHINE_MODE=PRODUCTION
4

初始化資料庫

cd apps\api && pnpm prisma migrate deploy && pnpm prisma db seed
5

生成 OPC UA 憑證

pnpm exec node-opcua-pki createPKI
pnpm exec node-opcua-pki certificate --selfSigned -o client_cert.pem
6

更新機台 Endpoint

pnpm exec prisma studio  # http://localhost:5555
7

建置前端

pnpm --filter web build
8

啟動服務

pnpm --filter api start
9

開機自動啟動

npm i -g pm2 && pm2 start "pnpm --filter api start" --name cnc-scada && pm2 save
10

TNC640 Option 83 確認

MOD → Software Options → Option 83 = Active
MOD → Network/Remote Access → OPC UA NC Server → ON
